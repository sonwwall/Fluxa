## Markdown 代码块样式调整

- 目标：文章正文中的 Markdown 代码块只保留单层边框，避免 Shiki 高亮容器和 `pre` 包装层形成双框。
- 主要文件：
  - `fluxa-web/src/features/articles/components/markdown-components.tsx`：调整 `pre` 渲染逻辑和普通代码块单框样式。
  - `fluxa-web/src/features/articles/components/markdown-renderer.tsx`：调整 Shiki 代码块容器样式，内部 `pre` 只负责滚动和代码排版。

## Markdown 代码块双框修正

- 目标：修正外层 `pre` 仍然生成边框导致代码块出现双框的问题。
- 主要文件：
  - `fluxa-web/src/features/articles/components/markdown-components.tsx`：将外层 `pre` 改为无边框间距容器，未高亮代码块由 `code` 节点自身提供唯一边框。
  - `fluxa-web/src/features/articles/components/markdown-renderer.tsx`：保留 Shiki 代码块内容容器的唯一边框，内部 `pre` 透明化。

## Markdown 代码块复制按钮

- 目标：在文章代码块右上角增加复制按钮，点击后切换为对号状态提示复制成功。
- 主要文件：
  - `fluxa-web/src/features/articles/components/markdown-code-block.tsx`：新增客户端代码块组件，处理复制交互和图标状态。
  - `fluxa-web/src/features/articles/components/markdown-components.tsx`：普通代码块接入统一复制组件。
  - `fluxa-web/src/features/articles/components/markdown-renderer.tsx`：Shiki 高亮代码块接入统一复制组件。

## 行内代码嵌套错误修复

- 目标：修复行内代码被误判为代码块后生成 `<p>` 内嵌 `<div>`/`<pre>` 的 hydration 错误。
- 主要文件：
  - `fluxa-web/src/features/articles/components/markdown-components.tsx`：仅带语言的 fenced code 走代码块复制组件，普通 `code` 保持行内渲染。
