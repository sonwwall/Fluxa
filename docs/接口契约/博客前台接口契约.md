# 博客前台接口契约

本文档记录 Fluxa 第一阶段博客前台页面的前后端接口契约，覆盖公开首页、文章列表、文章详情、相关文章和作者介绍页。

当前前端相关 API 边界：

- `fluxa-web/src/features/articles/api/articles.ts`
- `fluxa-web/src/features/profile/api/profile.ts`

## 1. 范围

本阶段覆盖：

- 首页文章聚合：`/`
- 文章列表：`/articles`
- 文章详情：`/articles/[slug]`
- 详情页相关文章
- 作者介绍页：`/about`

暂不覆盖：

- 登录、注册和注册审批。
- 评论、订阅、搜索、项目页。
- 仅注册用户可见文章的完整访问流程。
- 网站统计接口。

## 2. 通用约定

- 接口路径使用 `/api/v1` 前缀。
- 响应体固定使用 `code`、`data`、`msg`。
- 成功响应 `code` 固定为 `0`。
- HTTP API 字段使用 `snake_case`。
- 前端 TypeScript 类型内部使用 `camelCase`，由 API 层转换。
- 时间字段使用 ISO 8601 字符串，例如 `2026-05-12T08:00:00Z`。
- 公开文章 URL 使用稳定可读的 `slug`。
- 第一阶段文章正文只支持 Markdown，不支持 MDX。
- 游客请求只返回 `visibility = "public"` 且 `status = "published"` 的文章。

成功响应：

```json
{
  "code": 0,
  "data": {},
  "msg": "success"
}
```

分页响应：

```json
{
  "code": 0,
  "data": {
    "list": [],
    "page": 1,
    "page_size": 20,
    "total": 0
  },
  "msg": "success"
}
```

用于首页文章卡片、文章列表和相关文章。

```json
{
  "id": "art_001",
  "slug": "designing-resilient-systems",
  "title": "Designing Resilient Systems: Patterns That Scale",
  "excerpt": "A practical look at boundaries, observability, and patterns that keep production systems understandable as they scale.",
  "category": {
    "id": "cat_systems",
    "slug": "systems",
    "name": "Systems"
  },
  "tags": ["Architecture", "Reliability"],
  "status": "published",
  "visibility": "public",
  "published_at": "2026-05-12T08:00:00Z",

  "cover": {
    "accent": "sky",
```


用于首页聚合展示。

```json

  "featured": {},
  "latest": [],
  "popular": [],
  "topics": ["Go", "Next.js", "Systems", "Notes", "AI"]
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `featured` | ArticleSummary | 是 | 首页主推文章。 |
| `latest` | ArticleSummary[] | 是 | 最新文章列表。 |
| `popular` | ArticleSummary[] | 是 | 热门文章列表；统计未完成时可按发布时间或后端默认策略返回。 |
| `topics` | string[] | 是 | 首页主题标签。 |

## 4. 接口定义

### 4.1 获取首页文章聚合

```text
GET /api/v1/articles/home
```

用于首页 `/`。

后端规则：

- `featured` 优先返回作者设置的主推文章。
- 如果没有主推文章，返回最新公开已发布文章。
- `latest` 默认返回 4 篇。
- `popular` 第一阶段没有统计数据时，可以返回后端默认排序结果，但字段必须保留。

### 4.2 获取公开文章列表

```text
GET /api/v1/articles
```

用于文章列表页 `/articles`。

查询参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `page` | number | 否 | `1` | 页码，从 1 开始。 |
| `page_size` | number | 否 | `20` | 每页数量，最大值由后端限制。 |
| `category_slug` | string | 否 | 无 | 按分区过滤。 |
| `tag` | string | 否 | 无 | 按标签过滤。 |

后端规则：

- 只返回 `status = "published"` 的文章。
- 游客只返回最终可见性为 `public` 的文章。
- 按 `published_at` 倒序排列。
- `category_slug` 不存在时返回空列表，不返回错误。

### 4.3 获取文章详情

```text
GET /api/v1/articles/{slug}
```

用于文章详情页 `/articles/[slug]`。

错误响应：

```json
{
  "code": 033001,
  "data": null,
  "msg": "文章不存在"
}
```

后端规则：

- 文章不存在、未发布、已撤回或游客不可见时，统一返回文章不存在，避免泄漏私有文章存在性。
- `content` 返回 Markdown 原文。
- 后端不得返回未经确认的 HTML。

### 4.4 获取相关文章

```text
GET /api/v1/articles/{slug}/related
```

用于文章详情页底部相关文章。

查询参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `limit` | number | 否 | `3` | 返回数量。 |

后端规则：

- 不返回当前文章自身。
- 优先同分区，其次同标签，最后按发布时间倒序补足。
- 当前文章不存在或游客不可见时，返回文章不存在。

### 4.5 获取公开作者资料

```text
GET /api/v1/profile
```

用于作者介绍页 `/about`。

查询参数：无。

## 5. 前端映射

| 当前前端函数 | 对应后端接口 |
| --- | --- |
| `getArticleHomeData()` | `GET /api/v1/articles/home` |
| `getPublicArticles()` | `GET /api/v1/articles` |
| `getArticleBySlug(slug)` | `GET /api/v1/articles/{slug}` |
| `getRelatedArticles(slug)` | `GET /api/v1/articles/{slug}/related` |
| `getAuthorProfile()` | `GET /api/v1/profile` |

## 6. 待确认项

- `read_time` 由后端计算还是作者手动填写。
- `views` 第一阶段是否真实统计，还是暂时返回空字符串。
- 首页 `featured` 是否需要作者后台显式配置。
- `popular` 排序在没有统计模块前使用什么默认策略。
- 文章目录 `toc` 由后端从 Markdown 解析，还是前端渲染时生成。
- 作者资料是否只有一个全局记录，还是允许多作者。
- 作者公开链接是否需要后端校验协议白名单。
