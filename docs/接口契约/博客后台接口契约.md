# 博客后台接口契约

本文档记录 Fluxa 第一阶段博客后台页面所需的接口契约，覆盖 `/author`、`/author/articles`、`/author/articles/new` 和 `/author/articles/[id]/edit`。

当前前端 API 边界：

- `fluxa-web/src/features/author/api/author.ts`

## 1. 范围

本阶段覆盖：

- 作者后台仪表盘。
- 作者文章列表。
- 新建文章草稿页面。
- 编辑文章草稿页面。
- 保存草稿、发布文章、撤回文章、删除文章。

暂不覆盖：

- 评论管理。
- 媒体库。
- 项目管理。
- 外观设置。
- 用户审批。
- 统计明细和来源 IP 分析。

这些入口当前只作为导航占位，不进入正式业务实现。

## 2. 通用约定

- 接口路径使用 `/api/v1/author` 前缀。
- 作者后台接口必须经过统一鉴权中间件。
- 后端必须独立校验作者权限，前端隐藏按钮不算权限控制。
- 响应体固定使用 `code`、`data`、`msg`。
- HTTP API 字段使用 `snake_case`。
- 时间字段使用 ISO 8601 字符串。

成功响应：

```json
{
  "code": 0,
  "data": {},
  "msg": "success"
}
```

未登录：

```json
{
  "code": 011001,
  "data": null,
  "msg": "请先登录"
}
```

无作者权限：

```json
{
  "code": 012001,
  "data": null,
  "msg": "无权限访问作者后台"
}
```

## 3. 数据结构

### 3.1 AuthorArticleRow

用于作者文章列表、仪表盘最近文章。

```json
{
  "id": "art_1001",
  "title": "构建一个现代化的 Go 微服务架构",
  "excerpt": "探索 Go 微服务架构的设计原则与实践",
  "category": {
    "id": "cat_go",
    "slug": "go",
    "name": "Go",
    "visibility": "public"
  },
  "tags": ["微服务", "架构"],
  "status": "published",
  "visibility": "public",
  "published_at": "2026-05-20T14:32:00Z",
  "scheduled_at": null,
  "updated_at": "2026-05-20T14:32:00Z",
  "views": "12.5K"
}
```

`status` 允许值：

```text
draft | published | scheduled | archived
```

`visibility` 允许值：

```text
public | registered
```

### 3.2 AuthorArticleDraft

用于新建和编辑文章页面。

```json
{
  "id": "art_1001",
  "title": "构建一个现代化的 Go 微服务架构",
  "excerpt": "探索 Go 微服务架构的设计原则与实践",
  "content": "# 构建一个现代化的 Go 微服务架构实践\n\n...",
  "category_id": "cat_go",
  "tags": ["Go", "微服务", "架构"],
  "status": "draft",
  "visibility": "inherit"
}
```

`visibility` 允许值：

```text
inherit | public | registered
```

其中 `inherit` 表示未单独设置文章可见性，后端发布或展示时继承分区可见性。

### 3.3 AuthorCategory

```json
{
  "id": "cat_go",
  "slug": "go",
  "name": "Go",
  "visibility": "public"
}
```

## 4. 接口定义

### 4.1 获取作者后台仪表盘

```text
GET /api/v1/author/dashboard
```

用于 `/author`。

成功响应：

```json
{
  "code": 0,
  "data": {
    "summary": {
      "greeting": "Welcome back, Arjun",
      "published_this_month": 4,
      "pending_drafts": 1
    },
    "stats": [
      {
        "label": "Published",
        "value": "4",
        "detail": "4 live now",
        "tone": "violet"
      }
    ],
    "recent_articles": [],
    "chart_points": [720, 540, 680]
  },
  "msg": "success"
}
```

说明：

- `recent_articles` 使用 `AuthorArticleRow[]`。
- `chart_points` 第一阶段可由后端返回默认统计序列。
- 更细的网站统计接口待统计功能确认后另行补充。

### 4.2 获取作者文章列表

```text
GET /api/v1/author/articles
```

用于 `/author/articles`。

查询参数：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `page` | number | 否 | `1` | 页码。 |
| `page_size` | number | 否 | `20` | 每页数量。 |
| `status` | string | 否 | 无 | 按文章状态过滤。 |
| `category_id` | string | 否 | 无 | 按分区过滤。 |
| `keyword` | string | 否 | 无 | 标题或摘要搜索。 |

成功响应：

```json
{
  "code": 0,
  "data": {
    "list": [],
    "page": 1,
    "page_size": 20,
    "total": 0,
    "overview": {
      "published": 4,
      "drafts": 1,
      "scheduled": 1,
      "archived": 1
    },
    "categories": []
  },
  "msg": "success"
}
```

说明：

- `list` 使用 `AuthorArticleRow[]`。
- `categories` 使用 `AuthorCategory[]`。
- 作者后台文章列表可返回所有状态文章。

### 4.3 获取新建文章默认草稿

```text
GET /api/v1/author/articles/new
```

用于 `/author/articles/new` 页面初始化。

说明：

- 该接口不创建数据库记录，只返回页面初始化数据。
- 真正创建记录由保存草稿接口完成。

成功响应：

```json
{
  "code": 0,
  "data": {
    "draft": {
      "id": null,
      "title": "",
      "excerpt": "",
      "content": "# Untitled article\n\nStart writing in Markdown...",
      "category_id": "cat_go",
      "tags": [],
      "status": "draft",
      "visibility": "inherit"
    },
    "categories": []
  },
  "msg": "success"
}
```

### 4.4 获取文章编辑草稿

```text
GET /api/v1/author/articles/{id}
```

用于 `/author/articles/[id]/edit`。

错误响应：

```json
{
  "code": 033001,
  "data": null,
  "msg": "文章不存在"
}
```

### 4.5 保存文章草稿

```text
POST /api/v1/author/articles
```

用于首次保存新文章。

请求体：

```json
{
  "title": "构建一个现代化的 Go 微服务架构",
  "excerpt": "探索 Go 微服务架构的设计原则与实践",
  "content": "# 构建一个现代化的 Go 微服务架构实践\n\n...",
  "category_id": "cat_go",
  "tags": ["Go", "微服务", "架构"],
  "visibility": "inherit"
}
```

成功响应：

```json
{
  "code": 0,
  "data": {
    "id": "art_1001"
  },
  "msg": "success"
}
```

### 4.6 更新文章草稿

```text
PATCH /api/v1/author/articles/{id}
```

请求体同保存文章草稿。

成功响应：

```json
{
  "code": 0,
  "data": {
    "id": "art_1001"
  },
  "msg": "success"
}
```

### 4.7 发布文章

```text
POST /api/v1/author/articles/{id}/publish
```

请求体：

```json
{
  "scheduled_at": null
}
```

说明：

- `scheduled_at = null` 表示立即发布。
- `scheduled_at` 有值表示定时发布，文章状态进入 `scheduled`。
- 后端发布时计算最终可见性：文章单独设置优先，否则继承分区可见性。

成功响应：

```json
{
  "code": 0,
  "data": {
    "id": "art_1001",
    "status": "published"
  },
  "msg": "success"
}
```

### 4.8 撤回文章

```text
POST /api/v1/author/articles/{id}/withdraw
```

成功响应：

```json
{
  "code": 0,
  "data": {
    "id": "art_1001",
    "status": "archived"
  },
  "msg": "success"
}
```

### 4.9 删除文章

```text
DELETE /api/v1/author/articles/{id}
```

成功响应：

```json
{
  "code": 0,
  "data": null,
  "msg": "success"
}
```

## 5. 前端映射

| 当前前端函数 | 对应后端接口 |
| --- | --- |
| `getAuthorDashboard()` | `GET /api/v1/author/dashboard` |
| `getAuthorArticles()` | `GET /api/v1/author/articles` |
| `getNewAuthorArticleDraft()` | `GET /api/v1/author/articles/new` |
| `getAuthorArticleDraft(id)` | `GET /api/v1/author/articles/{id}` |
| `getAuthorCategories()` | 可由文章列表/草稿接口内联返回，或后续拆为 `GET /api/v1/author/categories` |

## 6. 待确认项

- 作者后台鉴权未登录时前端跳转登录页，还是展示 401 页面。
- `archived` 是否作为“已撤回”的后端状态名，还是使用 `withdrawn`。
- 定时发布是否第一阶段实现，还是只保留接口字段。
- 文章标签是否第一阶段作为简单字符串数组，还是独立标签表。
- 作者文章删除是否软删除。
- 文章保存和发布的表单校验规则。
