# 博客项目前期开发准备文档

本文档用于记录博客项目前期开发阶段的设想与待确认事项，作为项目启动前的头脑风暴和后续设计依据。

## 1. 项目目标

- 实现一套完整的博客系统，覆盖前端、后端、测试、部署等环节。
- 项目规模较小，整体采用单体架构。
- 在保证博客基础功能完整的前提下，探索更现代、更有创新性的前端展示和交互方式。

## 2. 用户角色与权限

项目计划支持三种角色：

### 2.1 游客

- 查看对外公开的博客文章。

### 2.2 注册用户

- 查看公开博客。
- 查看仅对注册用户开放的私人博客。

### 2.3 作者

作者对整个博客系统拥有完全控制权限，包括：

- 管理博客文章：发布、删除、修改、保存草稿等。
- 管理所有用户，例如删除用户、审批注册申请。
- 查看网站统计信息，例如流量、访问次数、来源 IP 等。
- 管理博客分区及其可见范围。
- 编辑作者介绍页面。

## 3. 功能规划

### 3.1 博客文章管理

- 作者可以在线编辑文章。
- 编辑器支持 Markdown 编写和预览。
- 标题、简介和正文均提供独立编辑区域。
- 文章时间自动写入。
- 文章支持以下状态：
  - 草稿
  - 已发布
  - 定时待发布
  - 已撤回，可再次编辑并发布

### 3.2 博客分区与内容可见性

- 博客可划分为多个分区，例如技术、日常、吐槽等。
- 作者可以自行创建和管理分区。
- 每个分区可以限定开放范围：
  - 游客可见
  - 仅注册用户可见
- 单篇文章可以单独调整可见性。
- 未单独设置文章可见性时，默认继承所属分区的可见性。

### 3.3 用户注册与登录

- 用户注册时需要填写：
  - 邮箱
  - 密码
  - 昵称
  - 头像，可选
  - 注册原因
- 默认情况下，注册申请需要作者审批后才能通过。
- 作者可以在后台关闭注册审批功能；关闭后，用户无需填写注册原因即可完成注册。
- 已登录用户使用 JWT Token 保持登录状态：
  - 前期使用单 Token。
  - 后期可升级为双 Token，并使用 Redis 管理登录状态。

### 3.4 作者介绍页面

- 提供独立的作者介绍页面。
- 页面可以展示作者介绍、相关链接等内容。
- 作者可以在后台编辑页面，所有展示元素均可替换。

### 3.5 网站数据统计

- 为作者提供网站统计页面。
- 计划统计的数据包括：
  - 网站流量
  - 访问次数
  - 来源 IP

## 4. 系统架构与技术选型

### 4.1 后端

- 基础技术栈：
  - Hertz
  - GORM
  - MySQL
- 采用 `handler + service + dao` 三层架构。
- 使用依赖注入组织模块之间的依赖关系。
- 即使项目规模较小，也需要保持良好的分层和分包结构。

### 4.2 前端

- 计划使用的技术栈：
  - Next.js
  - React
  - TypeScript
  - HeroUI
  - Tailwind CSS
  - Shiki，用于代码语法高亮
- UI 组件库候选：[HeroUI](https://github.com/heroui-inc/heroui)
- 提前定制统一的 Markdown 渲染样式。
- 可考虑支持 MDX。

## 5. 前端设计方向

- 前端整体风格需要统一、现代。
- 博客最重要的部分之一是内容展示，因此需要重视视觉设计与阅读体验。
- 可以使用 GPT Image 生成设计参考图，再由 Codex 根据参考图实现前端。
- 可参考 [Marvis 官网](https://www.marvis.com/en/) 的视觉风格和交互创意。
- Marvis 的部分交互方式可能不完全适合博客，需要结合博客的阅读场景进行调整和创新。

## 6. 代码与测试规范

- 前后端均需要保持良好的分层、分包架构。
- 后端每个文件应有对应的测试文件。
- 每个处理函数或独立功能尽量放在单独文件中。
- 避免单个文件过于臃肿。
- 项目根目录需要编写 `AGENTS.md`，用于规范 AI 的工作流程。

## 7. 部署规划

- 使用 Docker Compose 部署项目。
- 条件允许时，搭建自动化 CI/CD 流水线。
- 可考虑在服务器部署 Git 仓库和 Drone，实现开发期间推送后自动部署。

## 8. 开发前待确认事项

- 确定前端整体架构后，再开始正式编写前端。
- 明确注册审批功能的默认开启状态及具体审批流程。
- 明确文章与分区可见性冲突时的处理规则。
- 确定是否在第一阶段支持 MDX。
- 确定网站统计功能的具体指标、数据来源和隐私处理方式。
- 确定 CI/CD 是否纳入第一阶段开发范围。
